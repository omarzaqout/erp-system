## 🏗️ البنية التحتية والأساسيات (Core & Architecture)

تم التركيز على بناء هيكلية قوية، متقدمة، وقابلة لإعادة الاستخدام (Reusable) تتوافق مع معايير الأنظمة الإدارية الحديثة (مثل Odoo و SAP):

1. **فصل الملفات (Strict Separation of Concerns):**
   * تم تحويل جميع الـ Components في المشروع ليحتوي كل منها على 3 ملفات منفصلة: `.ts` (للمنطق)، `.html` (للهيكل)، و `.scss` (للتصميم). تم إلغاء أي أكواد مضمنة (Inline) بالكامل.
   
2. **الصفحة الموحدة (`PageWrapperComponent`):**
   * تم إنشاء مكون ذكي `<app-page-wrapper>` ليعمل كحاوية (Wrapper) لجميع الصفحات الرئيسية في النظام.
   * **فائدته:** يعرض عنوان الصفحة، مسار التصفح (Breadcrumbs)، القائمة العلوية للأزرار (Actions Header)، ويُطبّق تأثيرات حركية (Fade-in) بسلاسة. تم تطبيقه على كل الـ Modules، مما أدى إلى حذف مئات الأسطر المكررة وجعل تصميم النظام متناسقاً بدرجة 100%.

3. **أنظمة متقدمة (Advanced UI Systems):**
   * **نظام التبويبات (Tabs System):** يمكن للمستخدم فتح عدة نماذج أو صفحات في نفس الوقت كتبويبات (مثل متصفحات الويب)، مع ميزات إغلاق الجميع، التثبيت (Pin)، والقوائم المنبثقة (Context Menu).
   * **نظام النماذج الديناميكية (Dynamic Forms & Modals):** مخصصة لإنشاء `forms` بصورة ديناميكية داخل نوافذ منبثقة (`ModalFormComponent` و `SharedFormComponent`).
   * **الجداول المتقدمة (`DataTable`):** مجهزة بالترتيب، الترقيم (Pagination)، فلاتر البحث السريعة الموحدة، مع إمكانية تمرير أزرار التحكم بسهولة.

4. **التصميم والثيم (Theming & Styling):**
   * تم بناء ملف `_theme-config.scss` كمرجع أساسي للمتغيرات (Design Tokens).
   * النظام يدعم تغيير الثيم ديناميكيًا عبر `ThemeService`، واستخدام متغيرات CSS للتحكم في الألوان، المسافات، وحجم القائمة الجانبية.
   * تم ربط النظام بالكامل ليُدعم تعدد اللغات عبر نظام Pipes (`| translate`).

## 📦 تفاصيل الصفحات والأنظمة (Modules Breakdown)

### 1. 📊 لوحة القيادة (Dashboard)
* **الوظيفة:** النظرة العامة للمشروع.
* **المحتوى:**
  * بطاقات إحصائية (Stat Cards) لعرض الإيرادات، الطلبات، العملاء، والمنتجات منخفضة المخزون.
  * رسوم بيانية (Chart Placeholders) لعرض الإيرادات مقابل المصروفات.
  * قائمة "أحدث النشاطات" (Recent Activity).
  * قسم "أحدث الطلبات" مربوط بجدول بيانات، ورسم بياني صغير للمبيعات حسب الفئة القطاعية.

### 2. 🏪 نقاط البيع (POS System)
* **الوظيفة:** إدارة المبيعات المباشرة والكاشير بشكل احترافي.
* **المحتوى:**
  * **سلة المشتريات (Cart):** متطورة تشمل حساب الضرائب، الخصومات.
  * واجهة شاشة البيع (`PosContainer`).
  * تصفية المنتجات حسب الفئات، إضافة نظام المبالغ المستردة (Refunds) والتسعير بالجملة (Wholesale).
  * نظام تسجيل الجلسات المالي وتقارير الإغلاق اليومية (Z-Reports).

### 3. 🛍️ المبيعات (Sales)
* **الوظيفة:** إدارة طلبات العملاء والفواتير.
* **المحتوى:**
  * صفحة مخصصة للطلبات مع الفلترة والبحث السريع (`Orders`).
  * نموذج إضافة "طلب جديد" يعمل بشكل سلس يفتح داخل نظام التبويبات أو مباشرة.
  * صفحة مخصصة للفواتير (`Invoices`).

### 4. 🛒 المشتريات (Purchasing)
* **الوظيفة:** إدارة شراء المخزون وتوريده.
* **المحتوى:**
  * صفحة تتبع أوامر الشراء (`Purchase Orders`).
  * صفحة خاصة بإدارة الموردين (`Vendors`).

### 5. 📦 المخزون (Inventory)
* **الوظيفة:** تتبع وضبط المخزون والمنتجات.
* **المحتوى:**
  * صفحة إدارة المنتجات بطريقة متقدمة.
  * تتبع المستودعات المتعددة (`Warehouses`).
  * أزرار للفحص والمسح الضوئي (Scan Barcodes) المجهزة للربط البرمجي السريع.

### 6. 👥 الموارد البشرية (HR)
* **الوظيفة:** إدارة الموظفين والرواتب.
* **المحتوى:**
  * صفحة سجلات الموظفين وإضافة موظف جديد (`Employees`).
  * مسودة قسم لإعداد وإدارة الرواتب والأجور (`Payroll`).

### 7. 🤝 العلاقات العامة (CRM)
* **الوظيفة:** تتبع مسار المبيعات والعملاء المحتملين.
* **المحتوى:**
  * تصنيف العملاء المحتملين (`Leads`).
  * إدارة ملفات العملاء الكاملة (`Customers` & `Customer-Details`).
  * قسم إنشاء سجل عميل جديد مفصول ومُجهز للمتغيرات.

### 8. 💰 الحسابات والمالية (Accounting)
* **الوظيفة:** تتبع الدورة المحاسبية الكاملة.
* **المحتوى:**
  * شجرة الحسابات المنظمة (`Chart of Accounts`).
  * قيود اليومية للتعامل المالي (`Journal Entries`).
  * إدارة المعاملات المالية الموثقة (`Transactions`).
  * التقارير المحاسبية القياسية معدة شكليًا مثل "الميزانية العمومية" (Balance Sheet) و "الأرباح والخسائر" (Profit & Loss).

### 9. 📈 التقارير الشاملة (Reports)
* **الوظيفة:** استخراج التحليل النهائي المتقدم.
* **المحتوى:**
  * إحصائيات معدل التحويل (Conversion Rate).
  * تقارير بأكثر و"أهم المنتجات"، مع استعلامات وتصنيف "أفضل العملاء تعاملًا".

### 10. ⚙️ الإعدادات (Settings)
* **الوظيفة:** التحكم الشامل بإعدادات الشركة.
* **المحتوى:**
  * لوحة إعدادات ذات قائمة جانبية تتضمن: العام (General)، الشركة (Company)، الملف الشخصي (Profile).
  * إعدادات العملات وأسعار الصرف (`Currency Management`) تم ربطها مؤخراً لحفظ التغييرات.
  * تصميم تجهيزي للأمان التنبيهات والنسخ الاحتياطي وإدارة الـ Modules (تفعيل وتعطيل).

## 🛠️ الخطوات القادمة والمقترحات لمن سيستلم التطوير

1. **ربط الخلفية (API Integration):** البنية التحتية الجبهية (Frontend) واضحة وتستخدم الآن خدمة بيانات تجريبية (`MockDataService`). سيحتاج المبرمج لإحلال ملفات الـ HTTP Requests مكانها بسلاسة.
2. **إدارة الحالة (State Management):** لتوسيع النظام قد يكون من المفيد إدراج `NgRx` أو `Signals` في Angular لإدارة حالة (Cart) الخاصة بالبيانات، بيانات المستخدم وتسجيل الدخول، وأنظمة الفوترة المشتركة.
3. **الطباعة والتصدير:** تفعيل ميزات التصدير (PDF / Excel) الموجودة بالترويسة للرسوم البيانية والجداول.
4. **تفعيل الرسوم البيانية:** استبدال الأيقونات النائبة `Chart Placeholders` بمكتبة فعلية مثل `Chart.js` أو `Ngx-charts`.
